# 要件定義 & 画面設計

## 📱 画面遷移図 (Screen Flow)
このアプリはSingle Page Application (SPA)構成を採用しています。
HTMLファイルを分割すると画面遷移時に再読み込みが発生し、UX（アプリのような心地よさ）が損なわれるため、1つのHTML内でViewを切り替える設計としています。

1. **Dashboard (トップ画面)** `view: 'dashboard'`
   - ユーザー情報（Lv, EXP, Coin）表示
   - 週間活動グラフ
   - メニューボタン（ショップ、勉強開始など）
   - 管理者なら「Admin Mode」ボタン表示

2. **Studying (勉強中画面)** `view: 'timer'`
   - 経過時間表示（ストップウォッチ）
   - メモ入力欄
   - 終了/中断/取消 アクション

3. **Shop (ショップ画面)** `view: 'shop'`
   - 商品リスト表示
   - 購入モーダル

4. **Admin (管理者画面)** `view: 'admin'`
   - 親向けの管理メニュー（タスク追加、ポイント付与など）
